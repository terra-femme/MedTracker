# =============================================================================
# HEALTH-TECH NOTES FOR YOUR LEARNING:
# =============================================================================
#
# 🔒 PRIVACY-FIRST ARCHITECTURE:
#    - Ollama runs LOCALLY (no cloud API calls with patient data)
#    - SQLite stores data LOCALLY (no external database)
#    - Only drug NAMES go to RxNorm/OpenFDA (not patient info)
#
# 📊 VERSION PINNING:
#    - We use ">=" (minimum version) not "==" (exact version)
#    - This allows bug fixes while preventing breaking changes
#    - In production healthcare, you might pin exact versions for stability
#
# 🏥 DEPENDENCY CHAIN:
#    Your App
#       └── FastAPI (web framework)
#       └── LangGraph (AI workflows)
#              └── LangChain (LLM foundation)
#                     └── Ollama (local LLM)
#                     └── ChromaDB (vector storage)
#                     └── HuggingFace (embeddings)
#
# 🚀 WHAT EACH LIBRARY DOES IN YOUR APP:
#
#    USER TYPES: "Can I take ibuprofen with my blood pressure meds?"
#         │
#         ▼
#    [FastAPI] receives the HTTP request
#         │
#         ▼
#    [LangGraph] routes to "interaction_check" node
#         │
#         ▼
#    [LangChain + ChromaDB] retrieves relevant drug info
#         │
#         ▼
#    [Ollama + LangChain] generates the response
#         │
#         ▼
#    [LangGraph] adds safety disclaimers
#         │
#         ▼
#    [FastAPI] sends response back to user
#
